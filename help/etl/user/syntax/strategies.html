Transformation Strategies

To allow users incorporate transformation algorithms in Java, ETL provides the concept of transformation strategies. A transformation strategy is a class that extends the abstract class org.epsilon.etl.strategy.TransformationStrategy. Extending this class requires the definition of an autoTransform(Object source, EtlContext context) method that is responsible for transforming elements of the input model. As an example of a transformation strategy, org.epsilon.etl.strategy.emf.CommonEmfMetamodelStrategy creates an exact copy of the source model into the target model (useful in case of refinement transformations).

Overriding the transformation strategy

The transformation strategy is invoked for input elements when no transform-rule addresses them. Therefore, to override the transformation strategy for a specific type of elements one has to specify a transformation rule that addresses them.

Complementing the transformation stategy

Sometimes the functionality of a transformation strategy is very close but not identical to the desirable functionality for a specific problem. In such cases users can use the auto keyword to complement the functionality of the strategy for specific types of elements in the input model.
