Using ETL from Java


Here we provide exemplar code for using the ETL engine without Eclipse, but from a standard Java program. In this scenario we have the OO and DB metamodels (OO.ecore and DB.ecore), the instance models (OOInstance.ecore and DBInstance.ecore) and the ETL transformation (OO2DB.etl) n the folder "C:\EtlFromJava" (zip file containing all resources). To execute the transformation we define a java class Workbench with the code displayed here.

To run the following projects (or the respective jars) should be in the classpath:
  • org.epsilon.commons
  • org.epsilon.eol.engine
  • org.epsilon.eol.models.emf
  • org.epsilon.etl.engine
  • org.epsilon.erl.engine
Also, the following EMF-related jars must be in the classpath:
  • org.eclipse.emf.ecore.jar
  • org.eclipse.emf.ecore.xmi.jar
