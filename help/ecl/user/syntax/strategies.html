Matching Strategies

To allow users incorporate matching algorithms in Java, ECL provides the concept of matching strategies. A matching strategy is a class that extends the abstract class org.epsilon.ecl.strategy.MatchingStrategy. Extending this class requires the definition of an autoMatch(Object left, Object right, EclContext context) method that is responsible for comparing elements of the input models. As an example of a matching strategy, org.epsilon.ecl.strategy.emf.EmfIdStrategy compares model elemenets from EMF models based on the value of their xmi.id.

Overriding the matching strategy

The matching strategy is invoked for two input elements when no match-rule addresses them. Therefore, to override the matching strategy for a specific type of elements one has to specify a rule that addresses them.

Complementing the matching stategy

Sometimes the functionality of a matching strategy is very close but not identical to the desirable functionality for a specific problem. In such cases users can use the auto keyword to complement the functionality of the strategy for specific types of elements in the input models.
