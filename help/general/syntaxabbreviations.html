Syntax abbreviations

In this section we provide a list of abbreviations we use when presenting the syntax of Epsilon languages to minimize repetition.

block

A block is a sequence EOL statements.

blockOrExpression

Some constructs in Epsilon based languages need return a value. For example the compare part of a match rule must return true or false indicating whether the compared elements match. There are two options for computing this value: The first is to use the points character (:) followed by a single expression and the second is to use a block of EOL statements enclosed in curly brackets ({ }) that contains one (or more) return statements. Where we describe similar constructs, we represent this using the blockOrExpression abbreviation.

Syntax

(: expression) | ({ statement*})

Example

The following two fragments of code are equivalent:

compare : c.name = t.name

 and

compare {
  return c.name = t.name;
}
