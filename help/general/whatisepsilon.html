Introduction

Epsilon stands for Extensible Platform for Specification of Integrated Languages for mOdel maNagement. The purpose of Epsilon is to provide an integrated set of languages for the most common model management tasks such as model transformation, comparison, merging, validation etc and also to provide an infrastructure for building new task-specific languages with little effort and duplication. Currently, the following languages have been implemented in the context of Epsilon:
  • Epsilon Object Language (EOL)
  • Epsilon Transformation Language (ETL)
  • Epsilon Merging Language (EML)
  • Epsilon Comparison Language (ECL)
  • Epsilon Validation Language (EVL)

In this documentation the syntax and functionality of these languages are presented using concrete examples. To achieve uniformity and consistency in our examples we base them on three reference metamodels: one for Object Oriented systems, one for Databases and one for Type Mapping between different metamodels. The metamodels are provided in the KM3 language. KM3 is a language that abstracts above MOF and EMF and provides a textual syntax that is easy to write and to our view makes understanding metamodels easier than when expressed in UML-like diagrams. More information about KM3 can be found in http://www.eclipse.org/gmt/atl.
