ModelElement types

From an EOL module, many models of potentially different metamodels and technologies can be accessed/modified. To be able to identify models unambiguously, each model is assigned a unique name. Model element types can then be accessed using the ! operator. For example, in a UML 1.4 model named Uml, to retrieve a referecne to the Class model-element type, one has to use the Uml!Class expression. In case there are more than one model-element types with the same name (but e.g. in different packages) the fully qualified path of the type can be provided using the :: separator (in this case it would be Uml!Foundation::Core::Class). In case there is only one model accessible, the ModelName! prefix can be ommited.

Syntax

modelElementType

modelName!modelElementType

or

modelName!package1::package2::modelElementType

Built-in operations (on the type)

allOfType() : Sequence Returns a sequence that contains all the elements of the model with that type
allOfKind() : Sequence Returns a sequence that contains all the elements of the model that have a compatible type
allInstances() : Sequence Alias for the allOfKind() operation for OCL-users (can be used without the parentheses too)
createInstance() : Any Creates and returns a new instance of the type
isInstantiable() : Boolean Checks if new instances of the type can be created (e.g. in MOF, it checks that the meta-class is not abstract)

Examples

Uml!Class.createInstance();

Creates a new instance of Class in the Uml model.
Uml!Class.ofType(x);

Checks if x is of type Uml!Class

Built-in operations (on instances of the type)

id() : String Returns a unique id for the model element (e.g. in models loaded from XMI, it returns the value of the xmi.id property)
