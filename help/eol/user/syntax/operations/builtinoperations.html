Built-in operations

EOL provides a number of built-in operations on instances. There are two types of operations. Simple operations and operations declaring parameters. The latter apply only to collections. Documentation about the available built-in operations and their functionality is available for each EOL type separately (see the types subsection).

Overriding built-in operations

For convenience purposes built-in operations can be overriden by the user simply by specifying a user-defined operation with the same name. Even in this case, the build-in operation can be accessed by using the -> operator. An example of overriding a build-in operation appears here where the built-in println() operation is overriden so that instead of printing to the console it concatenates the source with a string buffer.
