. operator

In EOL, the . (dot) operator is used to access features or invoke operations on instances. In case an operation is called, the order in which candidate operations are examined is the following:

1. User defined operation
2. Built-in operation

For the reverse order, the -> (arrow) operator can be used.

Syntax

expression.featureCall

or

expression.operationCall(parameters)

Examples

a.println();

Invokes the built-in print() operation on instance a.
class.name

Returns the value of the feature name of instance class
