Introduction

The Epsilon Merging Language (EML) is a hybrid pattern-based model merging language. Using EML a user can merge two input models into a target model. All models involved in the process can be of different metamodels and technologies.

EML supports a 2-phase model merging process: In the first phase, matching elements are identified using match rules. If during the matching phase non-conforming elements are identified, the process stops and the elements are reported to the user. In the second phase, matching elements are merged using merge rules and non-matching elements are transformed into the target model using transform rules. EML re-uses the syntax and functionality of match rules and transform rules from ECL and ETL respectively.
