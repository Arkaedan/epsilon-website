Merging Strategies

To allow users incorporate merging algorithms in Java, ECL provides the concept of merging strategies. A merging strategy is a class that extends the abstract class org.epsilon.eml.strategy.MergingStrategy. Extending this class requires the definition of an autoMerge(Object left, Object right, EmlContext context) method that is responsible for merging elements of the input models. As an example of a merging strategy, org.epsilon.eml.strategy.emf.CommonEmfMetamodelStrategy merges EMF elemenets of the same metaclass automatically using the follwoing rules

  • For single valued features the value of the left element is preffered
  • For multi valued features the union of the values of both elements is used

Overriding the merging strategy

The matching strategy is invoked for two matchign input elements when no merge-rule addresses them. Therefore, to override the merging strategy for a specific type of elements one has to specify a merge rule that addresses them.

Complementing the matching stategy

Sometimes the functionality of a merging strategy is very close but not identical to the desirable functionality for a specific problem. In such cases users can use the auto keyword to complement the functionality of the strategy for specific types of elements in the input models.
